\section{Motivation}

Cloud Computing ist ein immer größer werdendes Thema in der IT. Immer mehr Daten werden nicht mehr lokal, sondern in der Cloud gespeichert und werden somit für jeden überall verfügbar. Auch Anwendungen werden immer häufiger nicht mehr lokal betrieben, sondern nutzen die großen Rechnerleistungen von Cloud Lösungen, um möglichst skalierbar zu sein und mittels Redundanz höhere Verfügbarkeiten und bessere Latenzzeiten zu erreichen. Zudem bieten Cloud Lösungen die Möglichkeit, Investitionskosten in Betriebskosten umzuwandeln, da keine neuen Server und andere Geräte gekauft werden müssen, sondern für die Nutzung der Cloud Services bezahlt wird. \newline Webapplikationen bilden hier keine Ausnahme. Anstatt eigene Rechenzentren aufbauen zu müssen, werden Webapplikationen immer häufiger bei externen Cloudanbietern betrieben um Kosten und den Administrationsaufwand solcher Systeme zu reduzieren. Ein großer Anbieter solcher Plattformen ist Amazon, die mit ihren Amazon Web Services \cite{aws} eine Reihe von Cloud Services bieten, die von vielen weltweit operierenden Unternehmen genutzt wird. \newline
Ein weiteres aktuelles Thema in der IT ist das Internet of Things und im Besonderen Smart Home. Es finden sich immer mehr Computer in Gegenständen des Lebens und erweitern deren Funktionen oder vereinfachen die Nutzung dieser Gegenstände. Dies können Geräte in der Industrie sein, die z.B. nicht mehr dauerhaft von Menschen betreut werden müssen, sondern selbst auf Veränderungen und Probleme in der Produktion reagieren oder zumindest einen Menschen alarmieren. Sie können auch untereinander kommunizieren, um Abläufe zu koordinieren oder andere Nachrichten auszutauschen. Hier wird häufig von Industrie 4.0 gesprochen bzw. der Informatisierung der Industrie, die die Effizienz der Produktion zukünftig erhöhen soll. \newline
Auch im Bereich Smart Home gibt es vielfältige Anwendungsmöglichkeiten. Hier finden sich Computer in Gegenständen des eigenen Hauses wieder, die in der Vergangenheit keine Computer beinhalteten. Beispielweise Rollläden oder Klimaanlagen könnten Teil des IoT werden und dann vom Nutzer mittels seines Smartphones auch außerhalb des Hauses gesteuert werden oder mittels Sensoren auf Umweltveränderungen reagieren. So könnten sich die Rollläden am Abend automatisch schließen oder Temperatursensoren könnten beispielsweise die Heizung anschalten, wenn es in der Wohnung zu kalt wird. Ebenfalls möglich wäre die Kommunikation mit dem Nutzer, der bspw. über sein Smartphone Nachrichten von Geräten des Hauses empfangen und dann darauf reagieren kann. \newline
Die beiden Themen Cloud Computing und IoT sind stark miteinander vernetzt, denn Geräte innerhalb des IoT sind meist mit der Cloud verbunden und können so beispielsweise untereinander kommunizieren oder über die Cloud mit dem Nutzer interagieren und gespeicherte Daten in der Cloud dem Nutzer übermitteln. Hier ist es gerade bei großen Datenmengen wichtig, dass die Daten auch bei hohem Datenaufkommen verlässlich in der Cloud persistiert werden und möglichst in Echtzeit überwacht werden können. \newline
In dieser Arbeit soll nun evaluiert werden, wie man mittels des Cloudservices Amazon Web Services große Datenflüsse wie beispielsweise die Aufzeichnung von Sensordaten in einem automatisierten Heimsystem am besten verwalten kann.

\section{Zielsetzung}
Es soll eine Wettersimulation erstellt werden, die aus mehreren Services besteht, die dauerhaft Daten liefern. Beispielsweise mehrere virtuelle Temperatursensoren, die Zufallszahlen innerhalb eines bestimmten Werteintervalls liefern. Die Temperaturwerte können sich auch gegenseitig beeinflussen. So hat bspw. eine höhere Außentemperatur auch Einfluss auf die Innentemperatur. Diese Komponenten werden jeweils in einem Docker Container \cite{docker} auf Amazon ECS \cite{ecs} deployed und liefern einen konstanten Datenstrom von Temperaturdaten. Der Datenstrom soll skaliert werden können und bspw. die Daten eines ganzen Tages in einer Stunde erzeugen. Die Daten werden in einer SQL Datenbank auf Amazon RDS \cite{rds} oder Amazon DynamoDB \cite{dynamodb} gespeichert. Eine eigene Lösung mit Apache Cassandra \cite{cassandra} wäre auch möglich. Im Dashboard soll eine Übersicht der Daten in Verlaufsdiagrammen verfügbar sein. Hier ist auch eine Anpassung der Zeitskalierung möglich. \newline
Der Fokus liegt hierbei nicht auf einer akkuraten Wettersimulation, sondern auf der Evaluation der AWS Services für große Datenströme und dem Vergleich zwischen den einzelnen AWS Services. Bspw. könnten Teile der Architektur auch mit Amazon Kinesis \cite{kinesis} umgesetzt werden, das speziell für große Datenströme konzipiert wurde. Hier könnte man beide Architekturentwürfe vergleichen und die jeweiligen Vor- und Nachteile herausstellen und so die passendsten Amazon Web Services für diese Aufgabenstellung ermitteln. Auch soll evaluiert werden, welche Datenbanklösung am besten geeignet ist und die beste Performance auch bei höherer Auslastung liefert. \newline
Dieses Projekt wurde in einer Projektarbeit bereits geplant und analysiert und soll in dieser Bachelorarbeit nun implementiert werden. Auf Basis dieser Implementierung soll eine Evaluation der verwendeten AWS Komponenten durchgeführt werden.

\section{Vorgehensweise}
Als erstes wird die Projektplanung zusammengefasst. Diese wurde bereits im Rahmen einer Projektarbeit \cite{projektarbeit} erarbeitet und muss daher nicht mehr ausführlich behandelt werden. Es wird nur kurz auf die einzelnen Planungsschritte und die Ergebnisse dieser eingegangen, um später Abweichungen im Projektverlauf im Vergleich zur Planung herausstellen zu können. Des Weiteren wird die Risikoanalyse nochmal kurz zusammengefasst, so dass später überprüft werden kann, ob Risiken eingetreten sind und ob die geplanten Gegenmaßnahmen gegriffen haben. \newline
Dann wird auf die Implementierung eingegangen, wobei zunächst die Projektarchitektur beschrieben wird, aus welchen Komponenten das Projekt aufgebaut ist und wie diese Komponenten arbeiten und miteinander in Verbindung stehen. Weitere Themen der Implementierung sind neben den Komponenten des Projekts auch das Deployment der Applikation. Hier wird zum einen die Einbindung in einen Docker Container als auch das Deployment auf AWS kurz beschrieben. Außerdem wird aufgezeigt, wie die erzeugten Ressourcen auf AWS wieder gelöscht werden können. Es wird zudem auf Abweichungen im Vergleich zur Projektplanung eingegangen. \newline
Im nächsten Schritt wird AWS IoT als weitere Alternative im Vergleich der im Projekt genutzten Amazon Web Services vorgestellt.  Hierbei wird gezeigt, wie auch echte Sensoren mit AWS verbunden werden können, welche Funktionen die IoT Lösung von Amazon bietet und welche Vor- und Nachteile AWS IoT aufweist. Zudem werden die von Amazon bereitgestellten SDKs und APIs kurz vorgestellt. \newline
Zuletzt werden die genutzten AWS Komponenten evaluiert und es wird auch auf den Nutzen von AWS IoT eingegangen. Zudem werden weitere Implementierungsmöglichkeiten und deren Vor- und Nachteile erörtert.
